## Constraints

- OSS core inviolable. Paid features live on top, never gate the canvas.
- Audience: hobbyist makers + educators. Price-sensitive.
- Solo dev hours. Infra cost must stay under 30% of revenue.
- Polar + better-auth already integrated (`@microflow/auth`).

## Conviction Split

### True (rally team here)
- Hobbyist + edu won't pay for desktop app itself (it's local-first, OSS).
- Paid value lives in **shared state** (collab, cloud sync, hosted broker) and **services** (LLM credits, education seats).
- Polar billing rail exists — no shopping cart to build.
- Bounded context `Identity & Billing` already in vision.

### Hypothesis (must validate)
- Hobbyists pay €5–10/mo for cloud sync between machines.
- Educators pay €20–50/mo for classroom seat pack.
- Designers pay for hosted MQTT broker (skip self-hosting).
- LLM credits convert better than BYOK as default.

## 80/20 Split

### Cannonballs (80% energy)
1. **Microflow Cloud** — single paid tier, €9/mo. Bundles: cloud flow sync, hosted MQTT broker, collab room persistence. Uses existing Yjs collab server + auth.
2. **Education Pack** — €29/mo for 1 teacher + 30 student seats. Pre-made lesson templates, classroom dashboard. Highest LTV per user.

### Lead bullets (20% energy)
- GitHub Sponsors + Polar one-time tip jar on docs site.
- Affiliate links to maker shops (Arduino, Adafruit, Tinytronics) on component docs.
- "Supporter" Polar tier €3/mo cosmetic (in-app badge, name in credits). Cheap WTP signal.

## Kill / Defer

- Template marketplace — too much UGC ops for solo.
- Self-hosted enterprise tier — wrong audience.
- LLM credit reseller as primary — BYOK simpler; revisit after Cloud lands.
- Per-feature paywalls on canvas nodes — violates OSS core.

## Sequencing (Q1 → Q3)

1. **Q1 — Validate WTP** (lead bullets first, cheap)
   - Sponsors/tip jar + Supporter tier live in 1 week
   - Landing page for Microflow Cloud (waitlist, capture intent)
   - Goal: 50 waitlist signups, 5 supporter subs
2. **Q2 — Ship Microflow Cloud MVP** (cannonball #1)
   - Cloud sync (Yjs persistence backend)
   - Hosted MQTT broker per workspace
   - Goal: 10 paid subs (€90 MRR)
3. **Q3 — Education Pack** (cannonball #2)
   - Classroom dashboard, seat invites, lesson templates
   - Outreach: 3 schools/coding clubs trial
   - Goal: 3 educator subs (€90 MRR) + 25 cloud subs (€225 MRR) = **€315 MRR**

## Reality Checks (Bavaro)

- Can solo dev ship Cloud MVP in Q2? Yjs persistence + Polar webhook = ~3 weekends. Yes.
- Education pack assumes classroom outreach skill. If no school contacts by month 5 → swap with hosted broker as standalone product.
- If Q1 lead bullets yield €0 → audience signal too weak; drop to donation-only model, reset target to €50/mo.
