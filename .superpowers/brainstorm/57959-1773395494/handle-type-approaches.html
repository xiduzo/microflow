Handle Type Legibility — Approaches

Four distinct directions for making handle types more readable. These can also be combined.

A — Always-on Color per Type

Keep the glyphs (●◆▶■), but each type has its own color — always visible, not just on hover or selection.

● value data flows
◆ event something happened
▶ trigger do this action
■ true boolean condition
✅ Minimal code change   ✅ Redundant encoding (shape + color)   ✅ Always readable

B — Precision CSS Shapes

Replace Unicode glyphs with crisp CSS-drawn shapes. More visually distinct, pixel-accurate, no font-rendering quirks.

value ● filled circle
event ◆ rotated square
command ▶ CSS triangle
state ■ rounded square
✅ Crisp at all sizes   ⚠️ More refactor (replace after: pseudoelements)   ✅ Scalable

C — Colored Type Badges

The handle label becomes a colored badge that shows both the name and the type. Vocabulary is always visible near the handle.

● value output
◆ event on change
▶ trigger input
■ true boolean out
✅ Self-documenting   ⚠️ More visual weight   ✅ Teaches the vocabulary

The wires between nodes take on the color of the handle type. The whole graph becomes a color-coded data flow diagram.
