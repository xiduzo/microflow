## H1 — Tutorial bypasses the MHB collection (correctness)

**File:** `learn/link-to-figma.mdx`

The tutorial tells the user to create a Figma variable called `pot_value` in any collection, then publish to `figma/pot_value`. The Bridge docs (`microflow-hardware-bridge/variables/index.mdx` and `microflow-hardware-bridge/variables/manipulating.mdx`) state that **only variables inside the `MHB` collection are connected to your hardware**.

Either the tutorial is wrong, or the Bridge docs are wrong. Both cannot be true.

**Fix:** Reconcile. Likely: amend the tutorial to instruct the user to use the auto-created `MHB` collection (and add a screenshot/image placeholder). Topic format becomes `figma/<variable-name-inside-MHB>`.

---

## H2 — "All five categories" math error

**Files:** `learn/index.mdx`, `learn/link-to-figma.mdx`

- `learn/index.mdx`: "When you finish all five, you have used every category of node in Microflow at least once." Microflow has **six** categories. The five tutorials cover Sense, Generate, Shape, Express, Internal — **Decide is never used**.
- `learn/link-to-figma.mdx`: "All five categories of node (sense, generate, shape, decide, express, internal — minus decide which gets covered in the Decide overview)" — lists six, claims five, then excludes one. Confusing.

**Fix:** Rewrite both to say "every category except Decide" and link to a new short tutorial (proposed L1 below) or the Decide reference index.

## H3 — Inconsistent node name: "MQTT (Express)" vs "Figma" vs "MQTT"

**Files:** `learn/link-to-figma.mdx`, `microflow-studio/nodes/express/figma.mdx`, `microflow-hardware-bridge/*`, multiple how-tos.

The publish-to-Figma express node is variously called:
- "MQTT (Express)"
- "Figma node"
- "MQTT node"
- referred to by path `express/figma`

`learn/link-to-figma.mdx` even apologises mid-step: *"its label says 'Figma' since it is the standard way to push values to the Bridge. (If your version names it `MQTT`, that is the same node.)"*

**Fix:** Pick the canonical name. Recommendation: **MQTT** (the protocol-accurate name; the node publishes MQTT, the Figma plugin is one subscriber). Update all prose; keep the file path for stable URLs. Add a redirect/alias note in the reference page header.
