Definitions for the terms that show up across these docs.

### Analog pin
A pin that can read a continuous range of voltages, typically `0`–`1023` on a 10-bit Arduino. Marked `A0`, `A1`, etc. on the board.

### Bridge
Short for [Microflow Hardware Bridge](/docs/microflow-hardware-bridge), the Figma plugin that connects MQTT messages to Figma variables.

### Canvas
The blank infinite grid in Microflow Studio where you place and connect nodes.

### Component
The runtime equivalent of a node. When the flow runs, each node on the canvas instantiates a *component* in the Rust runtime. Most Microflow users never need this distinction; it shows up if you read source code or [contribute](/docs/contributing).

### Debounce
Filtering out the rapid spurious changes a mechanical input emits when it actually transitions once. See [Debounce a button](/docs/how-to/debounce-a-button).

### Digital pin
A pin whose value is either HIGH (~5 V) or LOW (0 V). Numbered without a prefix on the Arduino: pin `2`, pin `9`, etc.

### Edge
The line connecting two nodes on the canvas. Carries values from the source node's output to the target node's input.

### Express
The category of nodes that produce real-world output: LEDs, servos, motors, piezos, MQTT publishes, etc.

### Firmata / StandardFirmata
The protocol Microflow uses to talk to a microcontroller over USB. StandardFirmata is the specific implementation that runs on the board. See [The microcontroller bridge](/docs/concepts/the-microcontroller-bridge).

### Flow
The whole document on the canvas — every node, every edge, every setting.

### Generate
The category of nodes that produce values without external input: constants, intervals, oscillators, triggers.

### Handle
The visible dot on a node where an edge can attach. Inputs are on the left, outputs on the right.

### Hot-swap
Editing a flow while it is running. Microflow applies changes immediately without restarting the connection.

### Internal
The category of nodes used to debug and document a flow: Monitor, Note. They do not affect what the flow does.

### MQTT
A lightweight publish/subscribe messaging protocol Microflow uses to connect Studio to the Hardware Bridge (and to anything else that speaks MQTT). See [Run your own MQTT broker](/docs/how-to/run-your-own-mqtt-broker).

### Node
A single building block on the canvas. Each one has settings, inputs, outputs, and a behaviour. Belongs to one of six categories: Sense, Generate, Shape, Decide, Express, Internal.

### PWM (Pulse Width Modulation)
A way to fake an analog output by toggling a digital pin extremely fast. Used to dim LEDs and run servos. PWM-capable pins on an Arduino Uno are pins 3, 5, 6, 9, 10, 11.

### Sense
The category of nodes that read from the world: buttons, sensors, switches, MQTT subscriptions.

### Shape
The category of nodes that transform values: Calculate, Function, Map, Smooth.

### Decide
The category of nodes that route or hold signals: Compare, Counter, Delay, Gate.

### Studio
Short for [Microflow Studio](/docs/microflow-studio), the desktop app for building flows.

### Topic (MQTT)
The string identifier a publisher and subscriber use to find each other. Microflow uses topics like `figma/light_level` to connect a flow value to a Figma variable.

### Yjs
The CRDT library that powers Microflow's real-time collaboration. See [Collaboration](/docs/concepts/collaboration).
