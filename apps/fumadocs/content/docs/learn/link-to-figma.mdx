The Microflow ecosystem has a second tool for designers: the **Hardware Bridge**, a Figma plugin that lets a Figma file react to physical inputs. In this lesson you will send the live reading from a potentiometer into a Figma variable and watch a design respond.

## What you need

- The flow from [Fade an LED with a potentiometer](/docs/learn/fade-with-potentiometer)
- A Figma account
- The [Microflow Hardware Bridge](https://www.figma.com/community/plugin/1373258770799080545) plugin installed in Figma

<Callout type="info" title="Video placeholder">
  60-second Figma-side screencast: open plugin → MHB collection appears → create variable → bind to rectangle width → watch it move as a knob turns on a real desk.
</Callout>

## Step 1 — Open the Hardware Bridge plugin

In Figma, run **Plugins → Microflow Hardware Bridge**. The first time you run it the plugin automatically creates a variable collection called **MHB** (short for Microflow Hardware Bridge) in your file.

<Callout type="info" title="Only MHB variables are exposed to hardware">
The Bridge listens **only** to variables inside the `MHB` collection. Variables in any other collection are ignored. This is by design — it keeps your design system variables out of the live data path.
</Callout>

Leave the MQTT defaults — the Bridge ships connected to a free public MQTT server out of the box. The plugin should report **Connected**.

## Step 2 — Create a variable in the MHB collection

1. Open Figma's **Local variables** panel.
2. Select the **MHB** collection.
3. Create a new variable inside it. Call it `pot_value`. Type: **Number**.
4. Set the default to `0`.

Bind that variable to *something visible* — for example, the width of a rectangle, or the opacity of a layer. Bind it the same way you bind any Figma variable.

## Step 3 — Send from Studio

Back in Microflow Studio, in your potentiometer flow:

1. Add an [**MQTT (Express)**](/docs/microflow-studio/nodes/express/figma) node.
2. Connect the Potentiometer's output to the MQTT node's input.
3. In the MQTT node, set the **topic** to `figma/pot_value`. The prefix `figma/` is what the Bridge listens to; the suffix must exactly match the variable name in the MHB collection (case-sensitive).

That is it on the Studio side.

## Step 4 — Watch Figma react

Twist the knob. The Figma variable `pot_value` should update live, and any layer bound to it changes in real time.

If nothing happens:

- Verify the Bridge plugin says **Connected**, not **Reconnecting**.
- Check that both Studio and the Bridge are pointed at the same MQTT broker (the [default server](/docs/microflow-hardware-bridge/mqtt) unless you changed it).
- Open a [Monitor](/docs/microflow-studio/nodes/internal/monitor) on the MQTT node's input to confirm values are flowing.

## What just happened?

Microflow Studio is publishing the potentiometer's reading to an MQTT topic. The Hardware Bridge plugin is subscribed to that topic and writes the value into the Figma variable. Figma redraws whenever a variable changes.
