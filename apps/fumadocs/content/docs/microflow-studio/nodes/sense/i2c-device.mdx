title: I2C Device
description: Read from and write to I2C sensors like SHT21, TCS34725, BME280, BMP280, MPU6050, BH1750, and other I2C peripherals.
---

The I2C Device node lets you communicate with sensors and peripherals over the I2C (Inter-Integrated Circuit) bus. I2C uses just two wires — SDA (data) and SCL (clock) — and supports many devices on the same pair of wires, each identified by a unique address.

Use it for sensors that don't have a dedicated node in Microflow: temperature/humidity sensors, colour sensors, accelerometers, light sensors, ADCs, and distance sensors. Pick a [device preset](#device-presets) to auto-fill the settings, or choose **Custom** and enter the address and register by hand.

## Inputs

- **Write** — send bytes to the device at the configured address and register. Accepts a single number or an array of numbers
- **Trigger** — perform a one-shot read from the device

Most sensors don't need either input: once the node is on the canvas it **streams** on its own (see [How reading works](#how-reading-works)). Use **Write** only to configure a device that needs it, and **Trigger** only for read-on-demand.

## Outputs

- **Value** — the data read from the device, formatted according to the **Output format** setting:
  - **Raw bytes** → an array of numbers, one per byte (e.g. `[0, 44, 1, 8]`)
  - **Unsigned int** → the bytes combined big-endian into one positive number
  - **Signed int** → the bytes combined big-endian as a two's-complement number (can be negative)

A raw sensor value is rarely the final number you want. See [Transforming values](#transforming-values) to turn it into °C, %, lux, or millimetres.

## Settings

Open the settings panel to configure:

- **Device preset** — choose a known sensor to auto-fill address, register, read length, and output format, or **Custom** for manual setup. The auto-filled values stay editable
- **Address** — the 7-bit I2C address of the device (0–127)
- **Register** — the register (command) the read points at (0–255)
- **Read bytes** — how many bytes to read back (1–32)
- **Output format** — how to interpret the bytes: Raw bytes, Unsigned int, or Signed int
- **Interval** — how often to read, in milliseconds. Each sensor has a floor set by its conversion time; reading faster than that returns stale or zero values

<Callout type="warn" title="Interval is shared by the whole board">
The read interval is a single global Firmata setting, so **every** I2C node on the board runs at the *slowest* configured interval. If one sensor needs 180 ms and another 20 ms, both run at 180 ms. The presets already pick a safe floor per sensor.
</Callout>

## How reading works

When you add an I2C node, Microflow arms a **continuous read** on the board: the firmware re-reads the register every interval and pushes the bytes back on its own. You do not poll — the **Value** output just updates.

Devices that need waking up or configuring first — clearing the accelerometer's sleep bit, powering on the colour sensor's ADC, putting the pressure sensor into continuous mode — are set up automatically for the presets that need it. The one exception is the **VL53L0X**, whose start-up is too involved for this to cover (see its entry below). For a **Custom** device that needs configuration, send the setup bytes to the **Write** handle once.

## Transforming values

The node hands you a raw number or byte array. To get a usable value, wire the **Value** output into a [Shape node](/docs/microflow-studio/nodes/shape):

- [**Function**](/docs/microflow-studio/nodes/shape/function) — write a formula in JavaScript. Best for datasheet conversions and for parsing raw byte arrays. The incoming value is `input`; `return` the result
- [**Map**](/docs/microflow-studio/nodes/shape/map) — rescale a number from one range to another (e.g. an ADC's 0–65535 to 0–100)
- [**Calculate**](/docs/microflow-studio/nodes/shape/calculate) — a single add/subtract/multiply/divide step
- [**Smooth**](/docs/microflow-studio/nodes/shape/smooth) — average recent readings to steady a noisy signal

Example — an **SHT21 (temp)** node outputs a raw 16-bit number. Wire it into a **Function** node to get degrees Celsius:

```ts
// SHT21/HTU21 raw temperature -> °C
if (typeof input !== "number") return 0;
const raw = input & ~0x0003; // clear the 2 status bits
return -46.85 + (175.72 * raw) / 65536;
```

For raw-byte sensors, `input` is the byte array — index into it inside a **Function** node:

```ts
// TCS34725 first 2 bytes = Clear channel, low byte first
if (!Array.isArray(input)) return 0;
return input[0] + input[1] * 256;
```

The [Device reference](#device-reference) below has a ready-made snippet for every preset.

## Device reference

What each preset sensor measures, the data it produces, and a [Function node](/docs/microflow-studio/nodes/shape/function) snippet to convert it. In every snippet the incoming reading is `input` and the converted value is `return`ed.

<Callout type="info" title="Ready vs. Advanced">
Sensors marked **Ready** stream usable data as soon as you add the node — Microflow runs any wake-up or configuration write the sensor needs automatically. You still convert the raw number to units downstream. Sensors marked **Advanced** need extra care the preset can't give you: the **VL53L0X** needs an external init sequence before it ranges at all, and the **BME280 / BMP280** stream only raw, uncompensated ADC counts that need per-chip calibration to become real units.
</Callout>

### SHT21 / HTU21 — temperature & humidity

**Address 0x40 · Ready.** A common humidity + temperature sensor. Two presets: temperature (register 0xF3) and humidity (register 0xF5). Microflow sets the sensor to 11-bit resolution and waits for the conversion automatically, so each preset just streams a raw 16-bit number.

**Data produced:** Unsigned int, 0–65535. The lowest two bits are status flags, not measurement — clear them before converting.

Temperature → °C:

```ts
// SHT21/HTU21 temperature
if (typeof input !== "number") return 0;
const raw = input & ~0x0003;
return -46.85 + (175.72 * raw) / 65536;
```

Humidity → %RH:

```ts
// SHT21/HTU21 relative humidity
if (typeof input !== "number") return 0;
const raw = input & ~0x0003;
return -6 + (125 * raw) / 65536;
```

Temperature and humidity are separate presets at the same address — add both nodes to read both. They share the bus fine.

### TCS34725 — RGB colour sensor

**Address 0x29 · Ready.** Measures clear, red, green, and blue light. Microflow powers on its ADC automatically.

**Data produced:** Raw bytes, 8 of them — four 16-bit channels in order **C, R, G, B**, each **low byte first** (little-endian).

Parse the channels:

```ts
// TCS34725 -> { c, r, g, b }
if (!Array.isArray(input) || input.length < 8) return 0;
const u16 = (lo, hi) => input[lo] + input[hi] * 256;
return { c: u16(0, 1), r: u16(2, 3), g: u16(4, 5), b: u16(6, 7) };
```

Normalise one channel to 0–255 relative to the clear channel (useful for driving an RGB node):

```ts
// TCS34725 red as 0-255
if (!Array.isArray(input) || input.length < 8) return 0;
const c = input[0] + input[1] * 256;
const r = input[2] + input[3] * 256;
return c === 0 ? 0 : Math.round((r / c) * 255);
```

### BH1750 — ambient light

**Address 0x23 · Ready.** A digital lux meter. The preset's register (0x10) is the "continuous high-resolution" command, so the sensor starts measuring on its own.

**Data produced:** Unsigned int, a raw 16-bit count.

```ts
// BH1750 -> lux
if (typeof input !== "number") return 0;
return input / 1.2;
```

### VL53L0X — time-of-flight distance

**Address 0x29 · Advanced.** A laser distance sensor. The preset reads the range result register (0x14) and reports millimetres directly.

**Data produced:** Unsigned int — distance in millimetres.

```ts
// VL53L0X -> centimetres
if (typeof input !== "number") return 0;
return input / 10;
```

<Callout type="warn" title="Needs initialisation">
The VL53L0X requires a long start-up sequence before it produces valid ranges. Microflow does not run that sequence yet, so use this preset only if your board or a prior sketch has already initialised the sensor. Readings of 0 or 8190 mean "not ranging".
</Callout>

### ADS1115 — 16-bit ADC

**Address 0x48 · Ready.** A 4-channel analog-to-digital converter for reading analog signals over I2C. Microflow configures it on startup for continuous conversions on single-ended channel **AIN0** at a **±4.096V** range, so the conversion register streams on its own.

**Data produced:** Signed int — a 16-bit conversion count, −32768 to 32767.

```ts
// ADS1115 -> volts (default range ±4.096V)
if (typeof input !== "number") return 0;
return (input * 4.096) / 32768;
```

<Callout type="info" title="Changing channel or gain">
The preset reads AIN0 at ±4.096V. To read a different channel or range, add a **Custom** node at 0x48 and send your own 16-bit config word to the config register (0x01) via the **Write** input, then adjust the `4.096` above to match the gain you pick. The [datasheet](https://www.ti.com/lit/ds/symlink/ads1115.pdf) config-register table lists the MUX and PGA bits.
</Callout>

### MPU6050 — accelerometer & gyroscope

**Address 0x68 · Ready.** A 6-axis motion sensor. The preset reads the 6 accelerometer bytes starting at 0x3B. The MPU6050 boots asleep, so Microflow clears its sleep bit automatically — the accelerometer axes stream on their own.

**Data produced:** Raw bytes, 6 of them — three 16-bit **signed, big-endian** axes: X, Y, Z.

```ts
// MPU6050 accelerometer -> { x, y, z } in g
if (!Array.isArray(input) || input.length < 6) return 0;
const s16 = (hi, lo) => {
  const v = (input[hi] << 8) | input[lo];
  return v >= 0x8000 ? v - 0x10000 : v;
};
const scale = 16384; // LSB per g at ±2g
return { x: s16(0, 1) / scale, y: s16(2, 3) / scale, z: s16(4, 5) / scale };
```

<Callout type="info" title="Reading the gyroscope">
The preset streams the accelerometer (register 0x3B). For the gyroscope, add a **Custom** node at address 0x68, register 0x43, 6 bytes — the same wake-up is already done, so it streams too.
</Callout>

### BME280 — temperature & humidity

**Address 0x76 · Advanced.** An environmental sensor. Two presets — temperature (0xFA) and humidity (0xFD). Microflow wakes the sensor into continuous (normal) mode automatically, so both stream on their own. For barometric pressure, use the **BMP280 (pressure)** preset below — the pressure register (0xF7) is the same on both chips.

**Data produced:** Unsigned int — the **raw, uncompensated** ADC reading, not a temperature or humidity.

<Callout type="warn" title="Raw ADC only">
Turning the BME280's raw ADC counts into °C / %RH / hPa requires each chip's factory calibration coefficients and the datasheet compensation formulas — more than a single Function node can do from one register. Treat these presets as raw. If you only need "hotter/colder" or "more/less humid", feed the raw value straight into a [Compare](/docs/microflow-studio/nodes/decide/compare) or [Smooth](/docs/microflow-studio/nodes/shape/smooth) node without converting.
</Callout>

### BMP280 — temperature & pressure

**Address 0x76 · Advanced.** The temperature-and-pressure sensor on the common purple breakout — the BME280's humidity-less sibling. Two presets: temperature (0xFA) and pressure (0xF7). Microflow wakes it into continuous (normal) mode automatically, so both stream on their own.

<Callout type="info" title="Address 0x76 or 0x77">
The presets use **0x76**. Some BMP280 breakouts tie the address-select (SDO) pin high, which puts the chip at **0x77** — if the node reads nothing, set its **Address** to 0x77 and try again.
</Callout>

**Data produced:** Unsigned int — the **raw, uncompensated** reading. Each preset reads 3 bytes; the measurement is the top 20 bits, so drop the low 4:

```ts
// BMP280 raw temperature or pressure -> 20-bit ADC count
if (typeof input !== "number") return 0;
return input >> 4; // the 3-byte read is 24 bits; the low 4 are unused
```

<Callout type="warn" title="Raw ADC only">
Like the BME280, the BMP280's raw counts need the chip's factory calibration coefficients and Bosch's compensation formulas to become real °C / hPa — more than a single Function node can do. Use the raw value directly with [Compare](/docs/microflow-studio/nodes/decide/compare) / [Smooth](/docs/microflow-studio/nodes/shape/smooth) for relative "warmer / higher-pressure" logic.
</Callout>

### Custom devices

**Address & register: yours.** Choose **Custom** to talk to any I2C chip. Set the address, register, read length, and output format from the datasheet.

- Find the register that holds the measurement and how many bytes it is.
- If the value is one number, pick **Unsigned int** or **Signed int** and convert with a [Function](/docs/microflow-studio/nodes/shape/function) node.
- If it's several fields, pick **Raw bytes** and parse the array in a Function node (see the MPU6050 and TCS34725 snippets for big-endian vs. little-endian).
- If the chip needs configuring first, send the setup bytes to the **Write** input once.

<Callout type="warn" title="Avoid clock-stretching commands on AVR boards">
On classic AVR boards (Uno/Nano/Mega) a "hold master" command that stretches the clock during conversion can freeze the whole bus. Prefer a "no-hold" command and set the node's **Interval** long enough for the conversion to finish.
</Callout>

## Wiring

I2C uses two shared signal lines plus power:

| Connection | From | To |
| --- | --- | --- |
| SDA | Sensor SDA pin | Arduino A4 (SDA) |
| SCL | Sensor SCL pin | Arduino A5 (SCL) |
| VCC | Sensor VCC | Arduino 3.3V or 5V (check your sensor's voltage) |
| GND | Sensor GND | Arduino GND |

<Callout type="warn" title="SDA and SCL are not interchangeable">
Swapping the SDA and SCL wires is the most common reason a sensor returns no data. Double-check them against the labels on your board and breakout before assuming the sensor is broken.
</Callout>

<Callout type="info" title="Multiple devices on one bus">
Connect as many I2C devices as you like to the same SDA/SCL lines, as long as each has a unique address. Two sensors that share an address (e.g. VL53L0X and TCS34725 both at 0x29) can't run at the same time unless one has an address-select pin. Microflow keeps every device on the bus streaming when you add or remove another.
</Callout>

<Callout type="info" title="Pull-up resistors">
I2C needs pull-up resistors on SDA and SCL (typically 4.7 kΩ to 10 kΩ). Most breakout boards include these already. A bare chip may need them added.
</Callout>
