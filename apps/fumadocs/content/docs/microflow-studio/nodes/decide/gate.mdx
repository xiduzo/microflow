A logic gate is a way to make decisions based on two inputs. Think of it like a simple rule that checks two conditions and gives you a result.

For example, an AND gate is like saying "both things must be true" - like "it's daytime AND the lights are off" before turning on a light. An OR gate is like saying "either thing can be true" - like "if it's dark OR someone presses a button, turn on the light."

Gates take two inputs (each either true or false) and produce a single output based on their logic. Here is a plain-English summary of each gate:

| Gate | What it means |
| --- | --- |
| **AND** | Both inputs must be true → output is true |
| **OR** | At least one input must be true → output is true |
| **NOT** | Flips the input — if input is true, output is false (and vice versa). Uses only one input. |
| **NAND** | Opposite of AND — output is false only when both inputs are true |
| **NOR** | Opposite of OR — output is true only when both inputs are false |
| **XOR** | Exactly one input must be true → output is true (if both are true, output is false) |
| **XNOR** | Opposite of XOR — output is true when both inputs are the same |

## Settings

Open the settings panel to configure:

- **Gate type** — select which logical operation to apply (AND, OR, NOT, NAND, NOR, XOR, XNOR)

Note: the **NOT** gate uses only one input — it simply flips whatever arrives. All other gate types use two inputs.
