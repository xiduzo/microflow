<Callout type="info" title="Also known as the &quot;Figma&quot; node">
Earlier releases labelled this node **Figma** because its most common use is publishing values to the [Hardware Bridge](/docs/microflow-hardware-bridge) Figma plugin. The label has been canonicalised to **MQTT (Express)** because the node speaks MQTT, not anything Figma-specific — the Figma plugin is just one possible subscriber.
</Callout>

The MQTT (Express) node publishes the value on its input onto an MQTT topic every time that value changes. Anything subscribed to the same topic — the Figma Hardware Bridge, the Penpot plugin, a Raspberry Pi, your own script — receives it.

**Example:** A potentiometer on your desk controls a slider in a Figma prototype. As you turn the dial, the value is published to `figma/<variable-name>`, the Bridge updates the matching variable in the `MHB` collection, and the slider moves in real-time.

This node pairs with the [MQTT (Sense)](/docs/microflow-studio/nodes/sense/mqtt) node, which subscribes to a topic.

## Setup with the Figma Hardware Bridge

1. Install and open the [Microflow Hardware Bridge plugin](https://www.figma.com/community/plugin/1373258770799080545) in your Figma file. The plugin auto-creates the `MHB` variable collection.
2. Create a variable inside the `MHB` collection (see [Connecting variables](/docs/microflow-hardware-bridge/variables/connecting)).
3. In Microflow Studio, add an MQTT (Express) node to your flow.
4. Set the **topic** to `figma/<variable-name>` — case-sensitive, must match the variable name exactly.

## Variable types

The node sends values in a format the Bridge can write into the target variable. The Bridge maps:

- **Number** — numeric values (e.g. a sensor reading of `750`)
- **Boolean** — `true` / `false` (e.g. whether a button is pressed)
- **String** — text
- **Color** — color values

## Settings

- **Broker** — which MQTT broker to publish to. Defaults to the shared public broker; change for self-hosted setups.
- **Topic** — the topic string. Use `figma/<name>` to drive the Figma Bridge or the Penpot plugin; any other topic for custom subscribers.
