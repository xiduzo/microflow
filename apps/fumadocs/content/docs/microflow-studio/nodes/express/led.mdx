An LED (Light Emitting Diode) is a small light that turns on when electricity flows through it. The LED node supports three variants: a single-color LED, an RGB LED, and an LED matrix.

**Example uses:**
- Turn on a light when a button is pressed
- Gradually dim a light based on a sensor reading
- Show status using different colors with an RGB LED
- Display patterns or animations on a grid of LEDs

---

## Single LED

A single LED can be switched on and off, or have its brightness controlled.

### Inputs

- **Value** — a number from 0 to 255 that sets brightness. 0 = off, 255 = full brightness. Values in between dim the LED
- **On** — turns the LED fully on
- **Off** — turns the LED fully off
- **Toggle** — switches between on and off

### Settings

- **Pin** — which pin on your Arduino the LED is connected to

<Callout title="Brightness requires a PWM pin">
Dimming an LED (sending values between 0 and 255) requires a PWM-capable pin. On an Arduino Uno, PWM pins are marked with a tilde (~): pins **3, 5, 6, 9, 10, and 11**.

A non-PWM pin can only switch fully on or off — sending intermediate values will have no effect. Use the On and Off inputs in that case.
</Callout>

---

## RGB LED

An RGB LED packs three LEDs — red, green, and blue — into a single component. Mixing the brightness of each channel produces any color.

**Color mixing reference:**
- Red 255 + Green 255 + Blue 0 = yellow
- Red 255 + Green 0 + Blue 255 = magenta
- Red 0 + Green 255 + Blue 255 = cyan
- All three at 255 = white
- All three at 0 = off

### Inputs

- **Red** — brightness of the red channel (0–255)
- **Green** — brightness of the green channel (0–255)
- **Blue** — brightness of the blue channel (0–255)

### Settings

- **Red pin, Green pin, Blue pin** — which PWM-capable pins each color channel is connected to (all three require PWM pins)

---

## LED matrix

An LED matrix is a grid of individual LEDs — typically 8×8 (64 LEDs total). You can display patterns, icons, or simple frame-by-frame animations.

You draw patterns using the built-in pixel editor in Microflow and create multiple frames. To display a specific frame, send its index number to the Show input.

### Inputs

- **Show** — which frame to display, as a number (0 for the first frame, 1 for the second, and so on)

### Settings

- **Pin** — which data pin the LED matrix is connected to
- **Frame editor** — draw pixel patterns for each frame directly in Microflow

<Callout title="Frames start at 0, not 1">
The first frame is frame 0, the second is frame 1, and so on. This is standard in programming — just remember to count from zero when sending values to the Show input.
</Callout>
