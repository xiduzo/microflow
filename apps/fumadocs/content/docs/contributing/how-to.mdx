---
title: How to contribute
description: How to report bugs, request features, and contribute code to the Microflow project.
---

Microflow is an open-source project — the code is publicly available and anyone can contribute. Whether you found a bug, have a feature idea, or want to submit a code change, here is how to do it.

## Found a bug?

First, check if someone else has already reported it: [search existing issues](https://github.com/xiduzo/microflow/issues).

If you cannot find your issue there, please [create a bug report](https://github.com/xiduzo/microflow/issues/new?assignees=&labels=&projects=&template=bug_report.md&title=). The more detail you include (what you did, what you expected, what actually happened), the easier it is for us to fix it.

## Want a new feature?

You can [request a feature](https://github.com/xiduzo/microflow/issues/new?assignees=&labels=&projects=&template=feature_request.md&title=) by opening an issue. Describe what you want and why it would be useful.

If you want to build it yourself, you are welcome to [open a pull request](https://github.com/xiduzo/microflow). Have a look at the existing code first to understand the project structure.
